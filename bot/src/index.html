Ki Detection & Timing
Mode
v2 tracks the shrinking ring and is the default. Use v1 only if Ring mode cannot read the chamber background reliably; it clicks after a fixed delay instead.
v1 · Click Delay
sec
v1 only: wait this long after a stable detect before clicking. Raise if clicks land too early; lower if too late.
v2 · Target Radius
× dot_r
v2 only: click fires when the tracked ring radius reaches this × dot_r. Lower = click LATER, when the ring is CLOSER to the dot. Higher = click EARLIER, ring still far. Tune live while watching in-game Perfect/Okay/Bad: if clicks land too early (ring still big at hit), lower it; if clicks land past Perfect (ring already vanished), raise it. Ring vanishes at ~1.40; below that you'll lose detection.
v2 · Latency Compensation
ms
v2 only: fire the click this many ms early using the ring's measured shrink speed. 0 = off (right for a fast PC on low ping). Raise if clicks consistently land late (ring already gone) on a slower PC or high-ping session — the ring shrinks ~10px per 100ms, so start around your ping.
v2 · Ring Brightness Min
v2 only: a pixel counts as "ring" if its max color channel ≥ this value. Default 220. Lower if the ring color is gray/dim; raise if bright backgrounds (white floors) are triggering false ring detection.
v2 · Ring Pixel Min
v2 only: minimum bright pixels (out of 32 sampled around each scanned radius) to call that radius "the ring". Default 6. Higher = stricter (avoids false detections from speckle); lower = more sensitive (catches partial / occluded rings).
Auto-save: changes apply immediately
